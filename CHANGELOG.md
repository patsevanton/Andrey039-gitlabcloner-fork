# Changelog

## v0.9

### Улучшения
- README дополнен документацией по опциям `Clone archived repos`, `Full clone`, `Checkout default branch before pull` — интерактивный пример, переменные окружения, поведение
- Поправлено форматирование выравнивания переменных в блоке `var` (`main.go`)

## v0.8

### Добавлено
- опция `Checkout default branch before pull` / `GITLAB_CLONER_CHECKOUT_DEFAULT_BRANCH` — автоматический checkout на ветку по умолчанию (main/master/develop) перед `git pull --ff-only`

## v0.7

### Добавлено
- пропуск архивированных проектов и клонирование репо только с последним коммитом
- опция `Full clone` / `GITLAB_CLONE_FULL` — полное клонирование вместо shallow clone (depth=1)
- опция `Clone archived repos` / `GITLAB_CLONE_ARCHIVED` — клонирование архивированных проектов

## v0.6

### Добавлено
- пропуск группы с ошибками вместо прерывания клонирования

## v0.5

### Добавлено
- Исключение проектов и групп по ID — настройка `Exclude IDs` / `GITLAB_CLONER_EXCLUDE_IDS` (через запятую)

## v0.4

### Исправлено
- Токен больше не сохраняется в `.git/config` склонированных репозиториев

### Добавлено
- Выбор протокола для origin: SSH (по умолчанию) или чистый HTTPS без токена — настройка `Origin protocol` / `GITLAB_CLONER_ORIGIN_PROTO`

## v0.3

### Добавлено
- Автоматический `git pull --ff-only` для уже склонированных репозиториев
- Вынесена общая функция `gitEnv()` для формирования окружения git-команд

## v0.2

### Улучшения
- `init()` заменён на `configure()` — инициализация вызывается из `main()`, а не неявно
- `cloneRepository` возвращает `error` вместо молчаливого логирования
- HTTP-ошибки от GitLab API теперь включают тело ответа для диагностики
- `strings.Replace` заменён на `strings.TrimPrefix` — безопаснее при обрезке путей
- Проверка ошибок `stdinScanner.Scan()` при чтении ввода
- `Clone dir` запрашивается последней при интерактивном вводе
- `GITLAB_CLONER_API_URL` переименован в `GITLAB_CLONER_API_PATH` — указывается только путь, не полный URL

### Добавлено
- `Makefile` с кросс-платформенной сборкой (`make build-all`)
- GitHub Actions workflow для автоматического создания релизов по тегу
- `README.md`

## v0.1

- Первая версия
- Рекурсивное клонирование всех репозиториев GitLab-группы
- Интерактивный ввод параметров и поддержка переменных окружения
- Пагинация через заголовок `Link`
- Пропуск уже склонированных репозиториев
